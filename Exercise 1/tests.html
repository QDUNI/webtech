Tests

How to pass?

You can only pass the course if you participate in the three partial exams (T1, T2, T3), and submit the three practical assignments on time (P1, P2, P3). Participation in lectures, seminars, or practical sessions is not compulsory, but staying away is not a good idea. All six components are assessed separately. At the end a total grade for the theory part is calculated: T = 0.2 * T1 + 0.3 * T2 + 0.5 * T3. A total mark for the practical part is also calculated: P = 0.2 * P1 + 0.4 * P2 + 0.4 * P3. If all parts are missing, your final mark is "ND" (not participated). You have not passed the course. If two partial exams or two practical assignments are missing, your final grade is "NVD" (not passed). You have not passed the course. If one partial exam and / or one practical grade is missing, your final grade is "AANV". You must then take an additional test and / or practical assignment to pass the course. If T < 5.00 or P < 5.00, the final grade E is the minimum of T and P. If T≥5.00 and P≥5.00, the final grade E is the average of T and P. The final grade is rounded to one decimal place, except between 5 and 6: the final grade is rounded to integers. So if E≥5.500 it becomes a 6, if E <5.500 it becomes a 5. If the final grade is at least 6, you have passed the course. If the final grade is lower than 4, you have not passed the course. If the final grade is 4,000 < E < 5,500, you can take an additional test and or practical assignment to try to pass the course. The additional test covers the entire material. The grade that you obtain for this test is entered for the missing grade, or for an insufficient partial grade of your choice. The mark for the additional practical assignment is entered for a missing or insufficient practical grade. The final grade is then recalculated according to the above rules. The mark for the additional practical assignment is entered for a missing or insufficient practical grade. The final grade is then recalculated according to the above rules. To participate in the additional test, the original result must be at least 4.

Tests

Here you can find the material that will be tested in the coming 3 exams.

Exam 1

  • Programs: Imperative programming to other paradigms (program consists of commands to change variables in memory) Compiler versus interpreter; edit-compile-run cycle Short Hello program with different approach (console, Forms, Game); structure of a program: `enumeration of classes, each class a bundle of methods, each method a bundle of assignments'. Syntax and semantics of method call Use of a development environment (currently Visual Studio)
  • Variables: Some methods of the Graphics class, as an example of using a standard class Use of variables, to avoid repetition of numerical constants Awarding contract, declaration, type int Layout and naming conventions Definition of new methods to abstract repeated blocks of code Definition of parameters to make methods more flexible
  • Interaction: The String class, conversion from int to String Structure of user-iterface with Button, TrackBar, TextBox) Event properties
  • Interaction: The String class, conversion from int to String Structure of user-iterface with Button, TrackBar, TextBox) Event properties

Exam 2

  • Choice: The if statement Applications of `if ': distinguishing from Buttons, checking texts, calculating minimum Case study: drawing and baseline parabola
  • Objects and classes: Class as description of an object, declaration of object variables Case: particle with location and speed as an object Simple animation with Thread and Sleep
  • Inheritance: Subclass: adding new variables and methods Difference between "is a" and "has a" Interface components as an example of class hierarchy User interface design with Layout managers Case: 4-function calculator

Exam 3

  • Strings and arrays: The type char versus class String Creation and use of strings Case: determining letter frequencies in a text (array as a peat table)
  • Object-oriented design: Abstract classes and interfaces Library classes for Collections Application: a vector drawing program Library classes for file I / O Non-window (console) programs; case: grep
  • The use of algorithms: Major case: a route search algorithm in the road network Major case: recognize the language of a text based on a comparison of letter frequencies (also an example of a "learning" program)
